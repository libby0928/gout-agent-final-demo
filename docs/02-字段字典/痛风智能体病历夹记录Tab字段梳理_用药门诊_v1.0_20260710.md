# 病历夹「记录时间轴」字段梳理（用药记录 Tab / 门诊记录 Tab）

> 版本：v1.0｜ 日期：2026-07-10  
> 适用范围：患者端「我的病历夹」页底部「记录时间轴」卡片   
> 术语：文中"字典行"指字段在《痛风专病智能体患者端字段字典 v1.1.1》中的行号，便于评审与开发直接定位原文。

---

## 一、整体框架（已与主任/产品确认）

病历夹底部「记录时间轴」卡片为**两个独立 Tab**，不是单一时间轴：

| Tab 顺序 | Tab 名称   | 默认状态     | 承载内容                                 |
| ------ | -------- | -------- | ------------------------------------ |
| 左（第 1） | **用药记录** | **默认选中** | 主视图：长期控制用药（每天规律吃的降尿酸药）；发作期对症药收进默认折叠区 |
| 右（第 2） | **门诊记录** | 非默认      | 门诊复诊节点 + 检查检验节点                      |

- **跨 Tab 顺序固定**：用药记录在左、门诊记录在右（纪要"用药靠前、门诊靠后"在 Tab 排布上体现为左→右）。
- **Tab 内部排序**：均按各自时间字段**倒序**（最近一条在最上方）。
- **门诊记录 Tab 加载方式**：纪要明确**上滑加载、不加分页按钮**；量大时用"上滑加载更多 / 已经到底了"轻提示，不打断时间连续性（提示文案与样式与用药 Tab 同源）。

---

## 二、页面整体展示顺序（病历夹整屏，从顶到底）

患者端「我的病历夹」一屏的信息架构顺序：

1. 档案摘要卡（姓名 / 性别 / 病种标签 / 关键指标概况）
2. 专病档案各分块入口（病史、用药、检验、检查、体格检查等，点击进入结构化详情）
3. 健康获得感知类模块（科普、票令查询、联系医生等，依首页状态规范）
4. **记录时间轴卡片**（本文件梳理对象）
   - Tab 栏：用药记录（默认）｜ 门诊记录
   - 用药记录面板（默认展开）
   - 门诊记录面板（上滑加载）

> 记录时间轴置于病历夹最底部，作为纵向历史的收口；已确认后的专病档案收纳于此，不在首页展开大块字段。

## 三、用药记录 Tab

> 设计目标：患者进入用药 Tab 第一眼看到"我现在在吃什么药"。**主视图只展示正在使用的长期控制用药**（每天规律吃的降尿酸药）；已停药收进底部「历史用药」折叠区，临时/急性期对症药收进「发作期用药」折叠区——两个折叠区默认收起，首屏只看在用药。

**必填 / 选填总览**

- **必填**：药物名称、状态标签（正在使用/已停药/按需）。缺任一项则这条药记录不成立、不该入轴。
- **选填**：用药方式+频次、每次用量+剂量、开始时间（缺失走 §3.1 降级链）、调整记录、用药备注、用药照片、停药原因、不良反应。
- **判定原则**：必填解决"识别这条是什么药、现在吃不吃"；其余都是增强信息（含怎么吃），缺失不应让卡片空白或报错。

### 3.1 时间字段（含缺失降级链）

- **主时间字段**：`用药日期`（字段字典 64 行）。
- **缺失降级链**（优先用前面的，没有才往后）：
  1. 患者填的`用药日期`（最准，首选）
  2. 医生确认 / 回写时间（患者未填，但医生在系统处理过这条药）
  3. 关联来源时间（该药随某次门诊 / 某份报告进入，借复诊日期或报告时间）
  4. 系统录入 / 创建时间（记录建出的时间戳，最次但有顺序）
  5. 均缺失 → **不进正式排序**，沉底并标"时间待补充"，不与有时间轴的药混排
- **历史调整条目**：用其调整发生日期；缺失按上述 2→5 链路回退。
- **原则**：时间缺失不影响展示，仅排序下沉 + 打标记；未"医生已确认"的记录本就不进正式时间轴（见§五），故"既没时间又没确认"的项沉底弱标记，不扰乱主记录。

### 3.2 主视图范围（只展示正在使用的长期控制用药）

- **主视图承载**：字典「长期降尿酸用药情况」（61 行）下、状态为"正在使用"的长期药，每天规律吃（苯溴马隆、非布司他等）。
- **不进主视图**：①已停药 → 收进「历史用药」折叠区（见 §3.7）；②急性期用药（71 行）下的临时/对症药 → 收进「发作期用药」折叠区（见 3.3）。
- **主视图默认状态**：展开即见正在使用的药，**首屏只看在用药**。
- 状态标签派生：正在使用 / 已停药（来自用药依从性 70）；长期药不标"按需"。

### 3.3 发作期用药折叠区（默认收起，临时药归此）

- **位置**：长期控制用药列表下方，一个默认折叠的「发作期用药 ▾」区块。
- **承载**：急性期对症药（止痛药、秋水仙碱、激素等按需用的），按用药日期倒序列出。
- **默认收起**：不占首屏，患者想看发作期吃过什么再展开；展开后单药卡结构与 3.4 相同（不含来源/确认标签，纯净展示）。
- 若用户无任何发作期用药，该折叠区不展示。

### 3.4 单药卡片字段（自上而下顺序，长期药与发作期药共用）

对齐字段字典「用药记录」分类（61–72 行）。必填＝缺了记录就识别不了"这是什么药、现在吃不吃"；选填＝有则展示、无则省略。

| 顺序 | 字段        | 字典行     | 必填     | 说明                                                |
| -- | --------- | ------- | ------ | ------------------------------------------------- |
| 1  | 药物名称      | 63      | **必填** | 主标题，缺失则记录无意义                                      |
| 2  | 用药方式 + 频次 | 65、66   | 选填     | "口服 · 每日一次"（长期药）；发作期药标"发作时按需"。有则展示，无则省略           |
| 3  | 每次用量 + 剂量 | 67、68   | 选填     | "1 片 / 50mg"，无则省略                                 |
| 4  | 状态标签      | 70 / 71 | **必填** | "正在使用 / 已停药"（长期）；"按需"（发作期）                        |
| 5  | 开始时间      | 64      | 选填*    | "开始于 2025-10"（长期药）；发作期药显示"最近使用"日期。缺失时按 §3.1 降级链回退 |
| 6  | 调整记录（详情内） | 64 + 摘要 | 选填     | 点开卡片后在详情区展示该药历次调整时间线（长期药重点；发作期药可简）                |
| 7  | 用药备注      | 69      | 选填     | "饭后服""和降压药隔开2小时"等服药情境，详情内展示                       |
| 8  | 用药照片      | 72      | 选填     | 拍药盒/处方留存，详情内缩略图入口（不进主视图）                          |
| —  | 停药原因与时间   | 待补（B 层） | 选填     | 「已停药」卡的补充：因皮疹/肝损伤等停，含停用日期。**P0**                  |
| —  | 不良反应记录    | 待补（B 层） | 选填     | 服药后皮疹/肝酶升高/胃肠道反应，与停药原因强相关。**P0**                  |

> \* 开始时间虽标选填，但**长期控制用药应尽量有日期**；缺失时按 §3.1 降级链处理，不强制阻断展示。
>
> **A 层展示遗漏（字典已有、双 Tab 未落，易补）**：依从性细分（70 行枚举含"规律/偶尔漏服/经常漏服"，当前只用"正在使用/已停药"，丢失了"在吃但经常漏"的高风险信号）→ 建议状态标签升级为"正在使用·规律 / 正在使用·偶漏 / 已停药"。**P1**。
>
> **待补（B 层）**＝字段字典当前未定义、需回填后才转为正式字段。一期 Demo 与最小实现不展示，待字典补齐且主任确认后落地。

### 3.5 展示示例（主视图在用药 + 历史用药折叠 + 发作期折叠）

```
用药记录 Tab（默认进入）：
  ── 长期控制用药（主视图，默认展开，按用药日期倒序）──
  ├ 苯溴马隆 · 正在使用 · 每日一次        （开始于 2025-10）
  ├ 非布司他 · 正在使用 · 每日一次        （开始于 2026-03，联合用药）
  ── 历史用药（1）▾（默认折叠，按停药时间倒序）──
     点开 → 别嘌醇 · 已停药 · 每日一次（2025-10 起，2026-05 因皮疹停用）
  ── 发作期用药 ▾（默认折叠，按最近使用倒序）──
     点开 → 依托考昔 · 按需 ｜ 秋水仙碱 · 按需

点开「苯溴马隆」卡（详情）：
  口服 · 每日一次 ｜ 1片/50mg ｜ 开始于 2025-10
  调整记录（治疗历程）：
    2025-10  开始苯溴马隆 50mg 每日一次（确诊后初始降尿酸治疗）
    2026-03  苯溴马隆单药尿酸控制不佳，加用非布司他 40mg 每日一次
    2026-06  复诊继续规律用药，等待下次血尿酸复查
```

> 主视图只展示正在使用的药，首屏干净；历史用药和发作期用药各自收进折叠区，患者按需展开。药多的患者不会被已停药卡片淹没。

### 3.6 来源（患者端不展示）

用药记录的来源维度为 患者自填 / 诊间采集（医生端同步）/ OCR / 历史，但**患者端不展示来源标签**（选项 A，见 §五）；来源信息保留在医生端用于追溯与确认流。

### 3.7 排序、加载与历史用药承载

- 主视图（长期控制用药）按**用药日期倒序**，只展示**正在使用**的药。
- 已停药收进主视图底部「历史用药 ▾」折叠区（默认收起），与「发作期用药 ▾」折叠区同一交互模式。
- 发作期用药折叠区内按用药日期倒序。
- 用药记录默认常规列表展示（纪要仅点名门诊记录用上滑，用药 Tab 不强制上滑）。

**历史用药承载方式：**

- **正在使用的药** → 主视图置顶，状态标签"正在使用"。
- **历史/已停用的药** → 收进「历史用药 ▾」折叠区，按停药时间倒序，状态标签"已停药"，并带用药起止时间（B 层待补：停药原因）。
- 折叠区标题显示数量，如「历史用药（3）▾」；无已停药时不展示该折叠区。
- 患者日常进用药 Tab 首屏只看到"现在在吃什么"，想回看换过什么药再展开折叠区。
- **跨药历史** = 折叠区里"已停药"状态的药（如别嘌醇），承载"换过什么药"。
- **单药内部历史** = 每张药卡点开详情后的"调整记录时间线"（如苯溴马隆 2025-10 起始 → 2026-03 加非布司他 → 2026-06 维持），承载"这一味药的完整历程"。
- 一期若停药原因（B 层）暂未实装，历史用药卡先显示"已停药 + 起止时间"，不显示"为什么停"；待字典回填后补"停药原因"行即可，不改承载结构。

## 四、门诊记录 Tab

门诊记录 Tab 内为**竖向时间轴**，含两类节点：**门诊记录节点**与**检查检验节点**。两类节点在同一时间轴上按时间混排、倒序。

**必填 / 选填总览**

- **门诊记录节点必填**：复诊日期、类型标签（系统带）、本次复诊摘要。缺任一项则节点不成立。
- **门诊记录节点选填**：门诊记录正文、复查项目、关联资料入口、就诊医院/科室/医生、诊断结论、发作情况、医嘱。
- **检查检验节点必填**：报告时间、类型标签（系统带）、报告名称、关键指标摘要。缺关键指标则报告无意义。
- **检查检验节点选填**：医院/科室、原图入口。
- **判定原则**：必填解决"哪天、什么就诊/报告、核心结论是什么"；其余为上下文增强，缺失不展示对应行即可，节点依然成立。

### 4.1 门诊记录节点

**时间字段**：`复诊日期`（字段字典 221 行）；缺失回退到医生保存/回写时间。

**展示字段**（节点内顺序）：

| 顺序 | 字段                | 字典行           | 必填     | 说明                                               |
| -- | ----------------- | ------------- | ------ | ------------------------------------------------ |
| 1  | 复诊日期              | 221           | **必填** | 节点顶部日期，缺失按时间回退规则处理                               |
| 2  | 类型标签              | "门诊记录"或"发作记录" | **必填** | 系统自动带，非数据字段；"发作记录"为门诊节点子类型，用于居家未就医的发作事件          |
| 3  | 本次复诊摘要            | 227           | **必填** | 节点标题，缺了患者不知道这次来看什么                               |
| 4  | 门诊记录（正文）          | 228           | 选填     | 节点描述/正文，简短就诊可空                                   |
| 5  | 复查项目              | 222           | 选填     | 有则显示，如"07-18 复查血尿酸、肝肾功能"                         |
| 6  | 关联资料入口            | —             | 选填     | 入口：查看化验 / 用药 / AI 依据（跳转对应详情）；无关联时不展示             |
| —  | 就诊医院 / 科室 / 医生    | 26-27、25（A 层） | 选填     | 跨院就诊上下文；字典归在"就诊关系"分类，门诊节点未显式落                    |
| —  | 诊断结论              | 91（A 层）       | 选填     | 本次诊断如"痛风性关节炎急性发作"；字典仅有检验下的"是否支持痛风诊断"，门诊节点无独立诊断字段 |
| —  | 本次发作情况（关节/VAS/红肿） | 186-188（A 层）  | 选填     | 急性发作就诊主诉核心；字典归在体格检查，不在门诊节点                       |
| —  | 医嘱 / 生活方式处方       | 字典无（A 层）      | 选填     | "低嘌呤、多饮水、戒酒"等；需补字典或归并门诊记录正文                      |
| —  | 发作诱因              | 待补（B 层）       | 选填     | 饮酒后/运动后/受凉/无诱因；复发预防核心。**P0/P1**                  |
| —  | 痛风石 / 结节进展        | 177、189（B 层）  | 选填     | 字典有超声痛风石(177)、体格检查痛风石(189)，但无节点承载进展              |
| —  | 手术 / 关节腔注射        | 待补（B 层）       | 选填     | 痛风石剔除术、关节腔抽液+激素注射。**P1**                         |
| —  | 在线问诊记录            | 待补（B 层）       | 选填     | 微信问医生"又痛了怎么办"，作门诊节点"在线问诊"子类。**P1**               |

> **A 层**（字典已有、展示遗漏）：就诊医院/科室/医生、诊断结论、发作情况可在不改动字典前提下直接补入门诊节点；医嘱/生活处方字典无独立字段，需补字典或归并正文。**P1**。
>
> **B 层**（字典无、需回填）：发作诱因、痛风石进展、手术/关节腔注射、在线问诊。一期不展示，待字典补齐且主任确认后落地。
>
> **发作事件**（P0）：患者居家突发红肿痛、吃依托考昔缓解、未就医——痛风最高频事件。**已确认：作为门诊节点「发作」子类**，不新增第三类时间轴节点，不动 §一 双 Tab 框架。门诊记录节点的类型标签除"门诊记录"外，增加"发作记录"子类型。

### 4.2 检查检验节点

**时间字段优先级**：`报告时间`（79）＞ `检查时间`（78）＞ `标本采集时间`（77）＞ 上传时间（系统记录）；即以报告单时间为准，无报告单时间则用上传时间。检查检验节点不论确认状态均进时间轴排序。

**展示字段**（节点内顺序）：

| 顺序 | 字段      | 字典行           | 必填     | 说明                                                |
| -- | ------- | ------------- | ------ | ------------------------------------------------- |
| 1  | 报告时间    | 79            | **必填** | 节点顶部日期，缺失按时间优先级回退                                 |
| 2  | 类型标签    | 固定"检查检验"      | **必填** | 系统自动带                                             |
| 3  | 报告名称    | 73            | **必填** | 如"生化检验报告单"；缺失则节点无法命名                              |
| 4  | 医院 / 科室 | 74、75         | 选填     | 元信息，有则展示                                          |
| 5  | 关键指标摘要  | 86+ / 85 / 82 | **必填** | 露 1–3 个核心指标，异常值标红；如"血尿酸 486 μmol/L（偏高）"。缺指标则报告无意义 |
| 6  | 原图入口    | 80            | 选填     | 点击查看报告原图；无原图时不展示                                  |

- 详情展开：点击节点进入报告单详情，按报告单类型展示具体指标（血尿酸、肝肾功能、尿常规、关节液等，见字典 86–185 行），异常/阳性指标优先展示。

### 4.3 排序、加载与锚点跳转

**排序**

- 两类节点在同一时间轴上**按时间倒序混排**（门诊记录用复诊日期，检查检验用报告时间，各自的优先级见上）。

**加载（纪要已定）**

- **上滑加载、不加分页按钮**：数据量大时增量加载，底部轻提示"上滑加载更多 / 已经到底了"，保持时间轴连续。

**锚点快速定位（A 类，在已有时间轴内跳转，非独立页）**

- **决策依据**：痛风患者复诊周期通常 1–3 个月一次，常需回看跨时间旧记录；纯上滑回看成本高、易迷失。故在守住"上滑无分页"前提下，补充轻量锚点跳转。
- **形式**：时间轴内按年/季度插入轻量分隔锚点（如年份小标 `2026` `2025`），点击直接定位到对应时间段；或上滑超过一定深度后浮现"选年份"悬浮锚，不常驻。
- **不做独立跳转页**：纪要明确门诊记录就在本 Tab 内上滑，再开独立页属架构冗余。
- **阈值建议**：预估节点 ≤20 条时锚点可不显；>20 条时启用。一期可预埋结构，按真实数据量决定。
- **视觉避让**：锚点不可做成吸顶大 Tab 栏，避免与顶部"用药记录 / 门诊记录"两个 Tab 视觉打架。

## 五、两块 Tab 共用的底座（来源 + 入轴规则 + 边界）

### 5.1 来源与入轴规则

**来源枚举**：患者自填 / 诊间采集 / OCR / 历史

**字段业务状态枚举**（字段字典通用状态）：  
暂无资料、患者自填、已提交、待医生确认、医生已确认、医生修正后确认、医生已忽略。

**入轴规则**：只有「医生已确认 / 医生修正后确认 / 回写成功」的内容进入时间轴正式排序；「待医生确认 / 已提交 / 患者自填」仅打状态标签，不混入正式记录时间顺序。

> **例外**：检查检验节点不论确认状态均进时间轴排序（检验报告本身是客观检查结果，不存在"医生还没认"的歧义，患者回看时需要完整检验历史）。

**来源展示策略（已定：患者端去来源、去确认环节，选项 A）**：

- **患者端不展示来源、不展示确认状态**。病历夹两块 Tab 的记录只呈现患者真正要用的信息，不出现"来源：XXX · 状态：医生已确认/待确认"这类标签。
- 理由：患者端展示确认状态会制造"我填的东西医生还没认"的焦虑；痛风患者要的是"现在在吃什么、上次血尿酸多少"，不是后台流转状态。来源与确认状态属于内部协作信息，留在**医生端**呈现。
- **保留的用药属性标签**：「正在使用 / 按需 / 已停药」仍展示——它描述的是"这药现在怎么吃"，不是确认环节。

### 5.2 两 Tab 字段边界明确清单

**用药记录 Tab —— 放什么**

- 主视图（长期控制用药，倒序，只展示正在使用）：药物名称、状态标签（必填）；用药方式+频次、每次用量+剂量、开始时间、调整记录时间线（选填）。
- 历史用药折叠区（默认收起）：已停药卡片，字段同单药卡 + 停药原因与时间、不良反应记录（B 层待补）；按停药时间倒序，无则不展示。
- 发作期用药折叠区（默认收起）：急性期对症药，字段同单药卡（不含来源/确认），无则不展示。
- **明确不放**：尿酸趋势曲线、检验数值汇总、门诊/就诊记录（归门诊 Tab）、合并症病史（归专病档案）。

**门诊记录 Tab —— 放什么**

- 门诊记录节点（含「发作记录」子类型）：复诊日期、类型标签、本次复诊摘要（必填）；门诊正文、复查项目、关联资料入口（选填）；就诊医院/科室/医生、诊断结论、发作情况、医嘱（A 层待补）；发作诱因、痛风石进展、手术/关节腔注射、在线问诊（B 层待补）。
- 检查检验节点：报告时间、类型标签、报告名称、关键指标摘要（必填）；医院/科室、原图入口（选填）。
- **明确不放**：尿酸跨节点趋势曲线与个体化目标线（另议）；合并症全貌（归专病档案）；用药记录（归用药 Tab）。
