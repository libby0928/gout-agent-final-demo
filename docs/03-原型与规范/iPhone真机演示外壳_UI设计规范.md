# iPhone 真机演示外壳 UI 设计规范

## 定位

iPhone 真机演示外壳用于把痛风智能体患者端、医生端或单个关键流程包装成“已经在手机里运行”的展示画面。它是对外演示容器，不替代业务 Demo，不承载完整业务规则，也不改变患者端或医生端主流程。

外壳的第一目标是真机感和聚焦感。画面只保留一台 iPhone、顶部模拟器控制条、手机内屏内容和柔和投影背景，不再使用开发窗口、代码编辑器、键盘、桌面图标或大段说明文字作为背景。

## 使用场景

适合用于汇报首页、Demo 封面、会议前预览页、产品截图页、短视频封面和单流程展示页。

不适合用于完整业务验收、医生端高密度工作台评审、流程联动测试或字段完整性检查。需要完整交互时，应打开原患者端或医生端 Demo。

## 页面结构

外壳页面固定由四层组成：

1. 浅色展示背景：用于衬托黑色手机机身，不能抢视觉焦点。
2. 手机投影：放在机身下方，模拟真机悬浮在桌面上的重量感。
3. iPhone 机身：黑色圆角外框、顶部灵动岛、右侧按键、屏幕玻璃反光。
4. 手机内屏：承载患者端或医生端的高保真展示内容。

页面不放左下角标题、不放营销式 Hero 文案、不放开发者说明、不放“演示外壳”“这里是预览”等内部口径。需要说明用途时写在文档里，不写进 UI。

## 画布与布局

桌面端画布使用全屏居中布局，最小高度 760px。手机主体水平居中，垂直位置略高于视觉中心，顶部给模拟器控制条留出空间。

iPhone 机身宽度建议在 390px 到 430px 之间。桌面端推荐 `width: min(430px, 84vw)`；移动端可降到 310px 到 330px。机身比例固定为 390:844，避免因内容变化拉伸变形。

顶部模拟器条宽度略小于或接近手机宽度，推荐 410px 到 430px，高度 42px 到 46px，向上贴近机身但不遮挡灵动岛。模拟器条只保留三色窗口点、设备名、系统版本和 2-3 个图标按钮，不扩展成工具栏。

手机内屏不出现第二层手机框。如果接入现有患者端 Demo，必须先抽取或重绘手机屏幕内部内容，不能直接把完整带机身的 HTML iframe 嵌入外壳。

## 颜色

背景采用低饱和浅灰蓝体系，推荐：

| 用途 | 颜色 |
|---|---|
| 页面顶部光感 | `rgba(255,255,255,0.9)` |
| 背景主色 | `#edf3f8` |
| 背景下沿 | `#cbd6e2` |
| 机身外框 | `#111827` |
| 灵动岛 | `#070b12` |
| 屏幕底色 | `#f3f7fb` / `#f7fbff` |

手机内屏沿用痛风智能体患者端的青绿到蓝色体系，重点色可使用 `#03d49f`、`#0184cf`、`#04a987`。不要把背景做成大面积深蓝、紫色渐变、炫光或开发环境暗色主题；这会削弱真机展示的干净感。

## 圆角与机身

iPhone 外框圆角建议 48px，内屏玻璃圆角建议 40px。外框 padding 固定 10px 左右，用于形成真实黑边。灵动岛宽度约 118px，高度约 34px，顶部距离约 17px，居中放置。

右侧按键可用 4px 宽、86px 高的小矩形模拟，位置在机身右侧上半段。按键不需要交互，不放 tooltip，不放文案。

## 阴影与反光

机身阴影使用两层：一层大面积柔和阴影表达悬浮，一层近距离阴影表达接触感。推荐：

```css
box-shadow:
  0 42px 88px rgba(15, 23, 42, 0.38),
  0 16px 34px rgba(15, 23, 42, 0.22),
  0 0 0 1px rgba(255,255,255,0.16) inset;
```

屏幕玻璃反光只作为轻微质感，透明度控制在 0.45 到 0.55，不能遮挡主要文字、按钮和关键数据。反光方向建议从左上到右下，避免横向白线穿过核心文案。

底部投影使用椭圆径向渐变和 blur，宽度约为机身宽度的 1.2 倍，颜色为低透明深蓝灰。

## 手机内屏内容

内屏内容必须像真实小程序首屏，不像网页截图。优先展示一个完整关键状态，而不是把所有功能塞进一屏。

患者端推荐展示“已纳管/诊前任务”状态，包括：顶部品牌、患者问候、任务卡、扫码就诊、专病档案摘要和底部导航。文案只保留患者当下要看的信息，如“补充痛风病史”“上传化验报告”“扫码就诊”“医生可查看已结构化资料”。

医生端如需放入同一外壳，应展示单一医生移动端状态或简化看板，不展示完整电脑端工作台。医生端高密度内容更适合平板或桌面外壳，不强行塞入 iPhone。

手机内屏不能出现开发说明、设计解释、字段 ID、演示步骤注释、占位说明、emoji 功能图标或“等待医生确认”作为主按钮。患者端统一使用“扫码就诊”，不写“扫码签到”。

## 字体与层级

外壳层使用系统字体：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue", sans-serif`。

手机内屏推荐层级：

| 层级 | 字号 | 字重 | 用途 |
|---|---:|---:|---|
| 顶部品牌 | 20px | 950 | 健康助手、页面主身份 |
| 大标题 | 28px | 950 | 当前核心任务 |
| 卡片标题 | 16px | 950 | 诊前任务、专病档案 |
| 列表主文案 | 14px | 950 | 任务名称 |
| 辅助说明 | 11px-13px | 700-800 | 来源、描述、状态 |

不要使用随视口缩放的字体。手机内屏文本必须在 310px 宽度下仍然不溢出、不遮挡、不压到灵动岛或底部导航。

## 交互规则

当前外壳可以先做静态高保真展示。若后续需要可点击，交互只发生在手机内屏，不让外壳本身变成复杂工具。

允许的交互包括：切换患者状态、展开任务详情、切换底部 Tab、播放 2-3 步轻量流程。禁止在外壳页面放右侧控制台、开发者导航、页面 ID 列表或演示说明按钮。

需要完整点击流程时，应跳转到正式 Demo 文件，而不是把完整 Demo 直接嵌进外壳。

## 与正式 Demo 的关系

正式业务页面仍以 `痛风智能体Demo设计规范_AI执行版.md`、患者端 HTML、医生端 HTML 和 `design-system/` 为准。真机演示外壳只抽取一个高价值状态做视觉包装。

当正式患者端页面发生主流程或关键状态变化时，外壳内屏文案和卡片也要同步更新，避免外壳展示的流程与真实 Demo 不一致。
