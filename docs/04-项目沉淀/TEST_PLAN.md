## 七、自动化测试脚本建议

### 7.1 推荐框架

由于当前 Demo 为纯静态 HTML，推荐以下测试方案：

| 层级 | 工具 | 用途 |
|------|------|------|
| 视觉回归 | Playwright Screenshot | 截图对比，确保页面样式不被破坏 |
| E2E 流程 | Playwright | 模拟用户操作流程，验证页面跳转和状态切换 |
| 文案审查 | 简单脚本（Node.js） | 全文搜索禁用词和敏感字段编码 |
| 字段校验 | Playwright | 验证表单校验逻辑（空值、格式错误等） |

### 7.2 Playwright 测试代码示例

#### 示例 1：快速建档必填校验

const { test, expect } = require('@playwright/test');

test('快速建档 - 姓名为空时不允许提交', async ({ page }) => {

  // 模拟进入快速建档页
  await page.click('[data-testid="btn-confirm-register"]');
  
  // 验证错误提示
  const errorMsg = await page.textContent('[data-testid="error-name"]');
  expect(errorMsg).toContain('请填写姓名');
});

test('快速建档 - 手机号格式错误', async ({ page }) => {
  await page.goto('...');

// test/text-audit.js
const fs = require('fs');
const path = require('path');

const FORBIDDEN_WORDS = [
  '扫码签到',

  'gout_history.',
  'patient.name',
  'patient.gender',
  'patient.mobile',
  'lab_report',
  'archive_status',
];

function auditFile(filePath) {
  const content = fs.readFileSync(filePath, 'utf-8');
  const issues = [];
  
  // 提取 body 中的文本内容（排除 style 和 script）
  const bodyContent = content.replace(/<style[^>]*>[\s\S]*?<\/style>/gi, '')
                             .replace(/<script[^>]*>[\s\S]*?<\/script>/gi, '');
  
  for (const word of FORBIDDEN_WORDS) {
    if (bodyContent.includes(word)) {
      issues.push(`禁用词: "${word}"`);
    }
  }
  
  for (const code of FORBIDDEN_CODES) {
    if (bodyContent.includes(code)) {
      issues.push(`字段编码泄露: "${code}"`);
    }
  }
  
  return issues;
}

## 十、跨端说明（患者端 ↔ 医生端）

本清单覆盖**患者端**测试。医生端具备患者端不涉及的能力，其测试见 [`医生端_TEST_PLAN.md`](./医生端_TEST_PLAN.md)，跨端逻辑基线见 [`INDEX.md`](./INDEX.md)：

- **医生端可主动建档（无码）**：工作站「+ 建档」填三字段直接建档案，不依赖患者扫任何码（ADR-D09，用例 TC-DR-07~10）。
- **医生端可事后拍照补录**：接诊后/跨日均可拍照补拍外院病历，OCR 直接入档，不依赖患者在线（ADR-D09，用例 TC-DS-06~08）。
- **弱接诊与兜底**：患者扫接诊码即资料可见，未点「接诊患者」也能采集/OCR/AI 辅助；未接诊患者留「待接诊」分组，超时触发「到院未接诊」预警（ADR-D01，用例 TC-DW-05~07、TC-DST-01、TC-DST-08）。

执行跨端回归时，以 [`INDEX.md`](./INDEX.md) 第二节逻辑基线为准，任一端改动涉及双端手机号/直接入档/弱接诊/建档流程时，须同步校验对端相关用例。
