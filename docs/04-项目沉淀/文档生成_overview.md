# 医生端四份项目文档生成 — Overview

## 任务
按用户确认的"新逻辑"（ADR-002 双端手机号 + ADR-003 患者/AI 内容直接入档、医生保留修正），将患者端已有的四份沉淀文档改写为**医生端视角**的完整版本。

## 产出文件（本次会话目录）
1. `医生端_PROJECT_SUMMARY.md` — 项目总览与架构：定位、7 大功能模块、11 屏 + 就诊码屏、ER 图、来源标记六类、技术栈、运行指南、已知限制。
2. `医生端_CLAUDE.md` — AI 开发规则：项目上下文、代码规范、字段规范（状态枚举已移除 confirm_status）、工作流、避坑指南（含视觉令牌红线）。
3. `医生端_PRD_决策日志.md` — 产品 PRD + 9 条 ADR（D01 弱接诊 / D02 双端手机号 / D03 直接入档 / D04 AI 治疗硬门槛保留 / D05 双码分离 / D06 来源六类 / D07 纯 HTML / D08 蓝色工作台 / D09 医生端主动建档+事后OCR）+ 验收标准。
4. `医生端_TEST_PLAN.md` — 测试验收：登录/工作站/建档/诊间采集/OCR/AI 辅助/逾期提醒流程、状态流转、字段校验、异常边界、文案审查、集成场景、Playwright 示例。

## 关键逻辑处理
- 沿用 ADR-002（手机号双端）、ADR-003（直接入档、医生可改）。
- **唯一保留的硬门槛**：AI 诊疗辅助的治疗建议（开药/调药/开检查）必须经医生确认执行——属医疗安全/医嘱执行门槛，与"内容入档"区分。
- 原有医生端源文档（产品报告/覆盖原则/降噪建议）中的"必须医生确认才入档"口径已按 ADR-003 修正为直接入档；字段确认页改为"核对/回写"页。

## 数据来源
- 医生端1.0产品报告.md、不同端口资料覆盖原则、两端交互断裂点反馈、信息降噪与层级优化建议、医生端Demo设计令牌_v1.0、病历夹 Tab 字段梳理。
- 患者端四份文档作为结构模板。
