# 资产索引

## 业务规则

`痛风智能体Demo设计规范_AI执行版.md` 是当前规则主入口。里面包含一期主线、患者端信息架构、医生端信息架构、视觉规范、组件复用规范、Demo 导航、状态联动、联系医生与患者群、待主任确认事项等内容。

`docs/不同端口资料覆盖原则_v1.0.0_20260707.md` 是不同资料入口之间的覆盖、确认和回写规则，包含患者端、医生端、OCR、录音提取和 AI 整理的主路径口径。

`AGENTS.md` 是协作规则入口。涉及主流程、页面信息架构、医生端/患者端关键状态、主任待确认事项、对外展示 Demo 的修改，需要先复述需求并等待确认。

`对话迁移说明.md` 是已确认上下文和迁移说明，适合在接手新对话或恢复上下文时先读。

## 可复用视觉

`design-system/tokens.css` 是视觉令牌。患者端主色是青绿到蓝色渐变，医生端主色是蓝色工作台体系；两端共用字号、圆角、阴影和间距变量。

`design-system/components.css` 是组件样式。它从当前患者端和医生端 Demo 中抽取稳定样式，不绑定具体业务页面。

`design-system/component-gallery.html` 是组件样例页，可直接打开查看患者端和医生端的基础组件。

`design-system/templates/iphone-showcase-patient.html` 是 iPhone 真机演示外壳场景模板。新做真机展示页时优先复制模板，再替换手机内屏内容。

## 图片素材

正式展示素材统一存放在 `assets/` 目录，并由 HTML 通过相对路径引用，避免外链、根目录散落文件或临时授权图片混入 Demo。

| 素材 | 用途 |
|---|---|
| `assets/doctor-qr.png` | 医生痛风专病建档二维码，供患者扫码建档、接诊码扫描演示、复诊扫码就诊和医生端“我的二维码”页面使用 |
| `assets/qr-icon.svg` | 二维码入口小图标，供医生端“我的二维码”快捷入口和菜单项使用 |

## 图标规则

Demo 页面不使用 emoji、符号或单字占位作为功能图标。导航、菜单、待办、OCR、复诊提醒等功能性图标统一使用页面内的 `.ui-icon` CSS 图形；头像姓名首字和流程步骤数字属于业务信息，可以保留为文字。

## 交互模式

患者端交互模式是“状态在页面上说清楚，任务在对话里完成”。病史补充、报告上传、复诊准备、健康打卡、饮食问答优先在首页对话容器内完成；扫码、联系医生、病历夹总览允许作为独立强任务。

医生端交互模式是“工作台给任务，接诊中采集，接诊后确认，确认后写入”。医生端不从完整档案开始，外层不突出问诊录音。
