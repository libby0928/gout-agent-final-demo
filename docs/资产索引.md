# 资产索引

## 业务规则

`痛风智能体Demo设计规范_AI执行版.md` 是当前规则主入口。里面包含一期主线、患者端信息架构、医生端信息架构、视觉规范、组件复用规范、Demo 导航、状态联动、联系医生与患者群、待主任确认事项等内容。

`AGENTS.md` 是协作规则入口。涉及主流程、页面信息架构、医生端/患者端关键状态、主任待确认事项、对外展示 Demo 的修改，需要先复述需求并等待确认。

`对话迁移说明.md` 是已确认上下文和迁移说明，适合在接手新对话或恢复上下文时先读。

## 可复用视觉

`design-system/tokens.css` 是视觉令牌。患者端主色是青绿到蓝色渐变，医生端主色是蓝色工作台体系；两端共用字号、圆角、阴影和间距变量。

`design-system/components.css` 是组件样式。它从当前患者端和医生端 Demo 中抽取稳定样式，不绑定具体业务页面。

`design-system/component-gallery.html` 是组件样例页，可直接打开查看患者端和医生端的基础组件。

## 可复用组件

患者端优先复用这些组件：

| 组件 | 用途 |
|---|---|
| `ds-patient-hero` | 未建档扫码主行动、建档完整度、已纳管复诊状态 |
| `ds-chat-card`、`ds-chat-message`、`ds-chat-bubble` | 阿福式 AI 对话和患者回复 |
| `ds-chip`、`ds-chip-primary` | 对话内选项和任务按钮 |
| `ds-question-card` | 推荐问题和快捷问法 |
| `ds-compose`、`ds-shortcut`、`ds-input-box` | 底部输入区和固定快捷入口 |
| `ds-record-card`、`ds-record-timeline` | 我的病历夹摘要和时间轴 |

医生端优先复用这些组件：

| 组件 | 用途 |
|---|---|
| `ds-doctor-head` | 医生信息头部和建档码入口 |
| `ds-stat-grid`、`ds-stat` | 工作台关键数字 |
| `ds-main-task` | 当前接诊主任务 |
| `ds-todo-list`、`ds-todo` | 医生待办 |
| `ds-status-chip`、`ds-source-badge` | 状态和来源标签 |
| `ds-field-card`、`ds-evidence` | 字段确认和依据展示 |

## 交互模式

患者端交互模式是“状态在页面上说清楚，任务在对话里完成”。病史补充、报告上传、复诊准备、健康打卡、饮食问答优先在首页对话容器内完成；扫码、联系医生、病历夹总览允许作为独立强任务。

医生端交互模式是“工作台给任务，接诊中采集，接诊后确认，确认后写入”。医生端不从完整档案开始，外层不突出问诊录音。

## 字段和字典

`痛风专病档案字段梳理.md` 是痛风专病档案字段表。患者端问法必须口语化，医生端保留结构化字段和确认逻辑。

`痛风相关检验检查字典.md` 是检验检查报告和指标字典。OCR 结果必须按报告/化验时间归档，写入档案前需要医生确认。
