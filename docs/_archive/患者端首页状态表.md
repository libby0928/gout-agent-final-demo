# 患者端首页状态表

本文档用于定义患者端首页在不同业务状态下应该展示什么。首页是患者进入痛风专病管理后的状态承载页，需要让患者一眼看懂三件事：我现在处于什么状态、还需要做什么、下一步从哪里进入。

本表覆盖患者端首页 7 类主状态。所有页面字段、按钮、数据行和提示文案均按本文档固定。

## 首页整体原则

| 原则 | 固定规则 |
| --- | --- |
| 首页核心目标 | 用一个稳定的首页状态区告诉患者当前管理状态，并给出下一步入口 |
| 卡片风格 | S3/S4/S5/S6-A/S6-B 统一使用同一套已纳管状态卡，只替换业务内容 |
| 底部快捷入口 | 始终保持「联系医生 / 上传报告 / 健康打卡 / 饮食问答」四个按钮，不因状态变化而增减、挪动或改样式 |
| 病历夹入口 | S2-S6 统一展示「我的病历夹」，放在状态卡右上角，不放到底部快捷入口中 |
| 扫码口径 | 患者端统一使用「扫码就诊」或「扫接诊码」 |
| 机器人消息 | 只说一句对患者有帮助的下一步建议，不重复卡片中的日期、倒计时、医生姓名、检查项目等结构化信息 |
| 无数据处理 | 没有数据的字段整行隐藏，不展示空值、横线或占位符；本文明确要求展示「暂无」的除外 |

## 首页状态判断顺序

系统按服务端最新状态判断首页展示。当多个条件同时满足时，按以下顺序命中第一个状态。

| 优先级 | 首页状态 | 业务含义 | 命中条件 |
| --- | --- | --- | --- |
| 1 | S5 逾期未复诊 | 患者已经超过建议复诊时间，但本次复诊还没有完成 | 已纳管，复诊日期已过，且本次复诊未完成 |
| 2 | S4 复诊当天 | 今天就是医生建议复诊的日期 | 已纳管，当前日期等于建议复诊日期 |
| 3 | S2 建档中 | 患者已提交基本信息，医生还没有完成确认 | 已提交快速建档，医生尚未确认回写档案 |
| 4 | S3 待复诊 | 医生已经安排未来复诊 | 已纳管，有未来复诊计划 |
| 5 | S6-A 确认无复诊需求 | 医生明确判断当前无需复诊 | 已纳管，医生明确标记无需复诊 |
| 6 | S6-B 暂无复诊安排 | 系统里暂时没有医生下发的复诊计划 | 已纳管，但无复诊计划，也无无需复诊标记 |
| 7 | S1 未建档 | 患者还没有进入痛风专病管理 | 无痛风专病档案，或未提交快速建档 |

患者正在就诊中时，首页展示「正在就诊」子状态，优先级高于 S3/S4/S5/S6-A/S6-B。就诊结束后，再根据医生是否下发复诊计划、是否标记无需复诊，回到对应首页状态。

## S1 · 未建档

S1 表示患者只是进入了小程序或打开了患者端首页，但还没有和医生建立痛风专病管理关系。页面重点是让患者知道：可以先问健康问题，也可以通过扫描医生建档码进入专病管理。

### 上方状态栏

| 字段 | 内容 |
| --- | --- |
| 状态标题 | 不展示 |
| 状态说明 | 不展示 |
| 状态标签 | 不展示 |
| 右上角入口 | 不展示 |

### 中间信息卡

不展示。

### 主行动 · 次行动

| 字段 | 内容 |
| --- | --- |
| 主行动 | 「扫码关联医生，进入痛风专病管理」——调起扫码，识别医生建档码后进入快速建档页 |
| 次行动 | 常见问题胶囊，最多 2 个：「痛风发作怎么办？」/「饮食要注意什么？」 |

### 下方机器人消息

「你好，我是你的健康助手。你可以先问我一些痛风相关的问题，也可以扫描医生的建档码，进入专病管理。」

## S2 · 建档中

S2 表示患者已经提交快速建档信息，但医生还没有完成确认和档案回写。患者已经进入痛风专病管理流程，但档案还不是正式已纳管状态。首页重点是鼓励患者继续补充痛风病史和化验报告，减少医生接诊时的信息采集压力。

### 上方状态栏

| 字段 | 内容 |
| --- | --- |
| 状态标题 | 「基本信息已提交」 |
| 状态说明 | 「可以继续补充痛风病史和化验报告，医生接诊时会看到这些内容。」 |
| 状态标签 | 「医生待确认」 |
| 标签颜色 | 蓝绿色 |
| 右上角入口 | 「我的病历夹」——进入病历夹总览页，展示已提交和医生已确认内容 |

### 中间信息卡

| 字段 | 内容 |
| --- | --- |
| 数据行 1 | 关联医生：`{doctorName}医生 · 痛风专病` |
| 数据行 2 | 档案完整度：`{archiveCompletion}%` |
| 数据行 3 | 待补资料：按未完成任务展示「痛风病史」「化验报告」；全部完成后隐藏 |

### 主行动 · 次行动

| 字段 | 内容 |
| --- | --- |
| 主行动 | 默认「补充诊前资料」——进入痛风病史逐项问诊流程，完成后引导上传报告；资料已完成或跳过后改为「扫码就诊」——调起扫码，识别医生接诊码后建立本次接诊关联 |
| 次行动 | 「给医生留言」——跳转独立联系医生页；联系医生能力未开通时隐藏 |

### 下方机器人消息

| 子场景 | 固定文案 |
| --- | --- |
| 建档成功后首次展示 | 「你好，张明，基本信息已提交。接下来可以补充痛风病史和化验报告，医生接诊时会看到这些内容。」 |
| 病史未补充 | 「先来补充一下你的痛风病史吧，医生接诊时可以参考。」 |
| 病史已提交、报告未上传 | 「病史已提交。如果手头有化验报告，可以先上传给医生参考。」 |
| 资料已完成或跳过 | 「资料已整理好。到院后可以扫描医生的接诊码完成就诊。」 |
| 患者已扫码就诊 | 「已完成扫码就诊，医生确认后会更新你的档案。」 |

## S3 · 待复诊

S3 表示患者已经完成建档并被医生纳管，医生也已经安排了未来复诊。首页要给患者稳定的复诊预期：什么时候复诊、还有多久、需要提前准备什么。

### 上方状态栏

| 字段 | 内容 |
| --- | --- |
| 状态标题 | 「等待复诊」 |
| 状态说明 | 「医生已为您安排下次复诊，建议提前完成复查准备。」 |
| 状态标签 | 「待复诊」 |
| 标签颜色 | 蓝绿色 |
| 右上角入口 | 「我的病历夹」——进入病历夹总览页 |

### 中间信息卡

| 字段 | 内容 |
| --- | --- |
| 数据行 1 | 下次复诊：`MM月DD日` |
| 数据行 2 | 剩余天数：`还有 N 天` |
| 数据行 3 | 复查项目：`{examItems}`；无复查项目时隐藏 |

### 主行动 · 次行动

| 字段 | 内容 |
| --- | --- |
| 主行动 | 临近复诊或有待补资料时展示「补充复诊资料」——上传化验单、补充近期发作记录和用药变化；否则不展示 |
| 次行动 | 「提前到院也可扫码就诊」——调起扫码，识别医生接诊码后建立本次接诊关联 |

### 下方机器人消息

「可以提前整理近期用药和发作情况，复诊时医生能更快了解你的变化。」

S4 表示今天就是医生建议复诊的日期。首页重点是让患者到院后完成扫码就诊，主行动必须清楚、直接。

| --- | --- |

| 状态说明 | 「今天是建议复诊日，到院后请扫描医生出示的接诊码。」 |
| 状态标签 | 「今日复诊」 |
| 标签颜色 | 琥珀色 |
| 右上角入口 | 「我的病历夹」——进入病历夹总览页 |

### 中间信息卡

| 字段 | 内容 |
| --- | --- |
| 数据行 1 | 建议复诊日期：`MM月DD日` |
| 数据行 2 | 复查项目：`{examItems}`；无复查项目时隐藏 |

### 主行动 · 次行动

| 字段 | 内容 |
| --- | --- |
| 主行动 | 「我已到院，扫码就诊」——调起扫码，识别医生接诊码后建立本次接诊关联 |
| 次行动 | 联系医生已开通时展示「联系医生」——跳转独立联系医生页；否则不展示 |

## 正在就诊子状态

患者在 S4、S5 或其他已纳管状态下完成扫码就诊后，进入「正在就诊」子状态。该子状态只表达本次接诊已经关联，不额外生成新的复诊结论。就诊结束后，再根据医生是否下发复诊计划、是否标记无需复诊，回到对应首页状态。

| 状态标签 | 「已扫码就诊」 |
| 右上角入口 | 「我的病历夹」——进入病历夹总览页 |

| 字段 | 内容 |

| 数据行 1 | 本次就诊：已关联医生接诊 |

## S5 · 逾期未复诊

### 上方状态栏

| 状态标题 | 「复诊已逾期」 |
| 状态说明 | 「原复诊日期已过，建议尽快联系医生确认新的复诊安排。」 |
| 状态标签 | 「已逾期」 |
| 标签颜色 | 红色 |
| 右上角入口 | 「我的病历夹」——进入病历夹总览页 |

### 中间信息卡

| 字段 | 内容 |
| --- | --- |
| 数据行 1 | 原复诊日期：`MM月DD日` |
| 数据行 2 | 已逾期：`N 天` |

### 主行动 · 次行动

| 字段 | 内容 |
| --- | --- |
| 主行动 | 「联系医生」——跳转独立联系医生页 |
| 次行动 | 「已到院？扫码就诊」——调起扫码，识别医生接诊码后建立本次接诊关联 |

## S6-A · 确认无复诊需求

S6-A 表示医生已经明确判断患者当前不需要复诊。它不是系统缺少复诊数据，而是有医生结论、有操作记录、有时间戳的管理状态。首页要稳定表达"当前无需复诊"，不再催患者预约，也不制造复诊待办。

### 上方状态栏

| 字段 | 内容 |
| --- | --- |
| 状态标题 | 「暂无复诊需求」 |
| 状态说明 | 「医生已确认当前无需复诊，日常有问题可以继续咨询或查看病历夹。」 |
| 状态标签 | 「无需复诊」 |
| 标签颜色 | 稳定绿色 |
| 右上角入口 | 「我的病历夹」——进入病历夹总览页 |

### 中间信息卡

| 字段 | 内容 |
| --- | --- |
| 数据行 1 | 复诊结论：「当前无需复诊」 |
| 数据行 2 | 不展示 |

### 主行动 · 次行动

| 字段 | 内容 |
| --- | --- |
| 主行动 | 不展示 |
| 次行动 | 联系医生已开通时展示「联系医生」——跳转独立联系医生页；否则不展示 |

### 下方机器人消息

默认隐藏。

## S6-B · 暂无复诊安排

S6-B 表示患者已经纳入痛风专病管理，但系统暂时没有医生下发的复诊计划，也没有医生标记"无需复诊"。它不是患者待办，也不是医生已经判断无需复诊，只是当前没有复诊安排记录。首页要克制表达，不引导患者去"确认复诊"，也不要展示"最近更新"。要引导患者去复诊。

### 上方状态栏

| 字段 | 内容 |
| --- | --- |
| 状态标题 | 「暂无复诊安排」 |
| 状态说明 | 「系统暂未记录医生下发的复诊计划，后续如有安排会同步到这里。」 |
| 状态标签 | 「暂无安排」 |
| 标签颜色 | 中性蓝绿色 |
| 右上角入口 | 「我的病历夹」——进入病历夹总览页 |

### 中间信息卡

| 字段 | 内容 |
| --- | --- |
| 数据行 1 | 复诊安排：「暂无复诊安排」 |

### 主行动 · 次行动

| 字段 | 内容 |
| --- | --- |
| 主行动 | 「联系医生」——跳转独立联系医生页，引导患者主动联系医生安排复诊 |
| 次行动 | 不展示 |

### 下方机器人消息

「如有不适或需要复查，可以联系医生安排复诊。」

## 状态切换规则

| 当前状态 | 触发动作 | 切换至 |
| --- | --- | --- |
| S1 未建档 | 患者提交快速建档 | S2 建档中 |
| S2 建档中 | 医生确认并回写档案，有复诊计划 | S3 待复诊 |
| S2 建档中 | 医生确认并回写档案，无复诊计划 | S6-B 暂无复诊安排 |
| S3 待复诊 | 到达复诊当天 | S4 复诊当天 |
| S3 待复诊 | 复诊日期过期且未完成复诊 | S5 逾期未复诊 |
| S4 复诊当天 | 患者扫码就诊 | 正在就诊子状态 |
| S5 逾期未复诊 | 患者扫码就诊 | 正在就诊子状态 |
| 正在就诊子状态 | 就诊结束，医生下发新复诊计划 | S3 待复诊 |
| 正在就诊子状态 | 就诊结束，医生标记无需复诊 | S6-A 确认无复诊需求 |
| 正在就诊子状态 | 就诊结束，无新复诊计划 | S6-B 暂无复诊安排 |
| S5 逾期未复诊 | 医生确认改期申请 | S3 待复诊 |
| S6-A 确认无复诊需求 | 医生下发复诊计划 | S3 待复诊 |
| S6-B 暂无复诊安排 | 医生下发复诊计划 | S3 待复诊 |
| S6-B 暂无复诊安排 | 医生标记无需复诊 | S6-A 确认无复诊需求 |

## 底部快捷入口固定规则

底部四个快捷入口是首页固定能力，不参与 S1-S6 状态变化。任何首页状态下都不允许调整这四个入口的样式、布局、功能、交互行为、页面位置、外观或点击响应。

| 固定入口 | 固定文案 | 固定业务含义 |
| --- | --- | --- |
| 联系医生 | 「联系医生」 | 进入医生联系或留言能力；未建档时提示先关联医生 |
| 上传报告 | 「上传报告」 | 上传化验报告、检查资料或历史资料 |
| 健康打卡 | 「健康打卡」 | 进入痛风健康记录、发作记录、用药记录等打卡能力 |
| 饮食问答 | 「饮食问答」 | 进入饮食相关 AI 问答 |

## 研发实现边界

| 边界 | 要求 |
| --- | --- |
| 可配置内容 | 日期、天数、医生姓名、检查项目、档案完整度、任务完成状态 |
| 不可配置内容 | 状态标题、状态说明、按钮文案、底部四个快捷入口 |
| 缺数据处理 | 没有数据的字段整行隐藏 |
| 新状态扩展 | 不允许前端自行新增状态，必须先更新本文档 |
| Demo 落地 | 修改患者端 Demo 时只能调整首页状态区和机器人消息，不改底部快捷入口 |
