## 一、决策风险（待主任确认）

### R-01 患者建档是否需手动选择/填写病种
- **描述**：当前原型倾向由医生端"痛风专病建档码"自动带出病种，患者只填姓名/性别/手机号。需确认真实门诊是否存在一码多专病、跨专病建档、或患者自选病种场景。
- **类别**：决策风险
- **概率**：高（未确认即悬空）
- **影响**：高（卡住快速建档表单与医患关联设计）
- **_owner**：主任（确认人）/ 鱼宝（跟进）
- **应对预案**：
  1. 确认前 Demo 不放"选择病种"为必填；
  2. 准备 A 方案（自动带出）与 B 方案（可选病种）两套低保真；
  3. 设确认锚点：下次主任沟通必问。
- **硬截止与降级预案（2026-07-13 补）**：
  - **锚点现状**：7-10 登记标注"下次主任沟通必问"，至 7-13 仍开放、无具体日期，已形成空转。
  - **硬截止建议**：设定具体沟通截止日（建议 ≤ 2026-07-17 周五收口会前），到期未确认则按**降级预案 A 自动带出**继续推进，不阻塞主线。
  - **降级 A（默认推进）**：建档码绑定单一专病、患者不手选，快速建档维持三字段；B 方案（可选病种）原型冻结为备选，不进一期开发。
  - **解锁影响**：R-01 卡住的 WBS 1.1/1.4（快速建档表单、医患关联）在降级 A 下可立即开工，无需等确认。
- **状态**：开放（已设硬截止，降级预案就绪）
- **关联任务卡**：WBS 1.1 / 1.4
- **登记日期**：2026-07-10｜**更新**：2026-07-13

### R-02 候诊自填是否为一期主路径
- **描述**：原型倾向患者扫码建档后、等待确认前自行补充病情/用药/检查；需确认真实流程是否在诊间由医生引导完成，决定诊间录音回填是并行还是后续路径。
- **类别**：决策风险
- **概率**：高
- **影响**：高（卡住病史补充交互与录音回填设计）
- **_owner**：主任 / 鱼宝
- **应对预案**：
  1. 锚点前只做不依赖确认的结构化档案展示（不依赖路径选择）；
  2. 准备候诊自填主路径原型 + 诊间录音回填备选原型；
  3. 明确"明天向主任确认"为硬锚点。
- **硬截止与降级预案（2026-07-13 补）**：
  - **锚点现状**：7-10 登记标注"明天向主任确认"，至 7-13 已逾期、仍开放、无结果，主线空转。
  - **硬截止建议**：同 R-01，建议 ≤ 2026-07-17 前完成确认；到期未确认则按**降级预案 A 候诊自填主路径**继续，诊间录音回填作为后续路径（不阻塞一期）。
  - **降级 A（默认推进）**：患者端维持候诊自填主路径（补充病史/上传报告/扫码就诊串联），规范保留"待确认"口径；诊间录音回填原型冻结为二期或并行评估项。
  - **解锁影响**：R-02 卡住的 WBS 1.2（病史补充交互）与录音提取域在降级 A 下可继续深化，不等待确认。
- **状态**：开放（已设硬截止，降级预案就绪）
- **关联任务卡**：WBS 1.2 / 录音提取域
- **登记日期**：2026-07-10｜**更新**：2026-07-13
