# 项目结构

## 根目录

根目录保留可直接打开或交付的 Demo 和核心规则文档。HTML 文件是可演示成品，Markdown / Word 文件是流程、规则、字段和设计说明。

| 路径 | 用途 |
|---|---|
| `痛风患者端_阿福式交互迁移.html` | 患者端主原型，承载阿福式交互、建档、病历夹和复诊闭环 |
| `医生端_信息降噪版Demo.html` | 医生端接诊采集原型 |
| `痛风智能体Demo设计规范_AI执行版.md` | 业务规则、交互规则、视觉规则、组件复用规则 |
| `痛风专病档案字段梳理.md` | 痛风专病结构化档案字段 |
| `痛风相关检验检查字典.md` | 检验检查报告和指标字典 |
| `医生端信息降噪与层级优化建议.md` | 医生端信息架构和接诊主线说明 |
| `对话迁移说明.md` | 已确认规则和迁移上下文 |
| `assets/` | Demo 使用的图片素材 |
| `design-system/` | 可复用设计组件和视觉令牌 |
| `docs/` | 仓库说明和复用说明 |

## design-system

`design-system/tokens.css` 保存患者端和医生端的颜色、圆角、阴影、字号等基础变量。`design-system/components.css` 保存可复用组件样式。`design-system/component-gallery.html` 是组件样例页，可以直接在浏览器打开查看。

这个目录不是新的业务 Demo，而是把现有 Demo 中已经稳定的视觉语言抽出来，方便后续页面复用。

## 忽略内容

`.gitignore` 排除了 `.claude/`、`.playwright-cli/`、`.tmp-lark-docs/`、`feishu_auth_qr.png` 和 `output/`。这些是本地工具状态、临时授权文件或运行备份，不作为项目源文件存放。
