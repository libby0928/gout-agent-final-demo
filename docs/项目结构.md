# 项目结构

## 根目录

根目录保留可直接打开或交付的 Demo 和核心规则文档。HTML 文件是可演示成品，Markdown / Word 文件是流程、规则、字段和设计说明。

| 路径 | 用途 |
|---|---|
| `痛风智能体患者端_2026年7月3号（会议后修订版）.html` | 患者端主原型，承载阿福式交互、建档、病历夹和复诊闭环 |
| `医生端_信息降噪版Demo.html` | 医生端接诊采集原型 |
| `痛风智能体Demo设计规范_AI执行版.md` | 业务规则、交互规则、视觉规则、组件复用规则 |
| `痛风专病智能体 demo-修改版.html` | 旧综合 Demo，保留作历史参考，不作为默认修改入口 |
| `痛风专病智能体 demo.html` | 更早期原始 Demo，保留作历史参考，不作为默认修改入口 |
| `痛风智能体_WorkBuddy标注模式试验稿.html` | 标注模式试验稿，除非明确要求继续试验，否则不作为主线文件 |
| `痛风智能体_真机开发现场演示外壳.html` | 真机展示外壳参考，不替代患者端主 Demo |
| `痛风专病档案字段梳理.md` | 痛风专病结构化档案字段 |
| `痛风专病档案字段完整版.md` | 字段补充参考，患者端研发默认优先读 `docs/02-字段字典/` 下的字段字典 |
| `痛风相关检验检查字典.md` | 检验检查报告和指标字典 |
| `医生端信息降噪与层级优化建议.md` | 医生端信息架构和接诊主线说明 |
| `患者报到业务复盘对痛风智能体的启发.md` | 外部业务复盘材料转化出的产品定位、一期主线和后续确认问题 |
| `对话迁移说明.md` | 已确认规则和迁移上下文 |
| `assets/` | Demo 使用的图片素材，包含医生建档二维码和二维码入口图标 |
| `design-system/` | 可复用设计组件和视觉令牌 |
| `docs/` | 仓库说明和复用说明 |

## docs

| 路径 | 用途 |
|---|---|
| `docs/01-产品需求/` | 产品需求和流程文档归档 |
| `docs/02-字段字典/` | 患者端字段、专病档案字段、检验检查字段归档 |
| `docs/03-原型与规范/` | Demo 规则、页面规范和审计材料归档 |
| `docs/99-索引/` | 仓库导航、字段索引、当前工作台账和版本索引 |

## design-system

`design-system/tokens.css` 保存患者端、医生端和真机演示外壳的颜色、圆角、阴影、字号等基础变量。`design-system/components.css` 保存可复用组件样式。`design-system/component-gallery.html` 是组件样例页，可以直接在浏览器打开查看。`design-system/templates/` 保存可复制的场景模板，例如 iPhone 真机演示外壳。

这个目录不是新的业务 Demo，而是把现有 Demo 中已经稳定的视觉语言抽出来，方便后续页面复用。

## 忽略内容

`.gitignore` 排除了 `.claude/`、`.playwright-cli/`、`.tmp-lark-docs/`、`feishu_auth_qr.png` 和 `output/`。这些是本地工具状态、临时授权文件或运行备份，不作为项目源文件存放。正式展示图片必须放在 `assets/` 并进入版本库。
