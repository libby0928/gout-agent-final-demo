# 痛风相关检验检查字段字典

## 1. 通用枚举

| 枚举名 | 值 |
|---|---|
| report_type | lab、imaging、ultrasound、dect、joint_fluid、urine、physical_exam、other |
| source_type | ocr、patient_input、his、doctor_input、audio_extract、system |
| confirm_status | pending_confirm、confirmed、corrected、ignored |
| abnormal_flag | low、normal、high、critical_low、critical_high、unknown |
| yes_no_unknown | yes、no、unknown |
| positive_negative_unknown | positive、negative、unknown、not_tested |
| unit_source | original、converted、calculated |
| display_scope | patient、doctor、both、backend |

## 2. 报告主表字段

| field_key | 中文名 | 类型 | 必填 | 枚举/格式 | 规则 |
|---|---|---|---|---|---|
| report_id | 报告 ID | string | 是 | - | 主键 |
| patient_id | 患者 ID | string | 是 | - | 关联患者 |
| report_type | 报告类型 | enum | 是 | report_type | - |
| report_name | 报告名称 | string | 否 | - | 原始报告名称 |
| source_type | 来源 | enum | 是 | source_type | - |
| hospital_name | 报告医院 | string | 否 | - | - |
| department_name | 报告科室 | string | 否 | - | - |
| specimen_type | 标本类型 | string/enum | 否 | serum、plasma、urine、joint_fluid、image、other | - |

## 15. 计算与互斥规则

| rule_key | 规则 |
|---|---|
| uric_acid_unit_conversion | serum_uric_acid 单位为 mg/dL 时，可生成 serum_uric_acid_converted_umol_l；换算系数 1 mg/dL = 59.48 umol/L |
| converted_value_keep_original | 换算值不得覆盖原始 value_numeric、unit、reference_range_text |
| report_time_for_trend | 趋势排序使用 report_time；report_time 缺失时使用 sample_time；sample_time 缺失时使用 upload_time |
| diagnosis_fields_need_confirm | gout_diagnosis_support、hyperuricemia_flag、has_tophus、renal_ultrasound_stone、has_kidney_disease、has_fatty_liver 写入正式诊断或既往史前需要 confirm_status=confirmed/corrected |
| item_original_name_required | source_type=ocr 或 his 时，original_item_name 必填 |
| ocr_low_confidence_pending | ocr_confidence < 0.8 或 item_confidence < 0.8 时，confirm_status 默认 pending_confirm |

| field_key | 中文名 | 类型 | 必填 | 枚举/格式 | 显示范围 |

## 19. OCR 到解读 Pipeline

| step | 输入 | 输出 | 规则 |
|---|---|---|---|
| 1_report_classification | original_file_id、ocr_raw_text | report_classification | 先判断报告类型；surgery_record、discharge_summary、prescription 不进入检验数值异常判断，只保留原文和摘要 |
| 2_metadata_extraction | ocr_raw_text | report_name、hospital_name、sample_time、exam_time、report_time、specimen_type | 缺失时允许为空 |
| 3_item_extraction | ocr_raw_text | original_item_name、value_numeric/value_text、unit、reference_range_text、abnormal_flag | 保留原始项目名和原始单位 |
| 4_item_mapping | original_item_name | standard_key | 使用项目别名表映射；未映射时 standard_key=unmapped，standardization_status=partially_standardized |
| 5_reference_parse | reference_range_text | reference_low、reference_high | 支持区间、上限、下限、阴性/阳性、+ 号等级；不可解析时保留 reference_range_text |
| 6_unit_normalize | value_numeric、unit、standard_key | converted_value | 仅生成换算字段，不覆盖原始值 |
| 7_abnormal_judgement | value_numeric/value_text、reference_low/reference_high、abnormal_flag | abnormal_flag、interpretation_level | 优先使用报告原始异常标记；无异常标记时用参考范围比较 |
| 8_domain_interpretation | standard_key、abnormal_flag、业务标签 | interpretation_summary | 按痛风相关业务标签生成摘要 |
| 9_review_gate | 置信度、缺失项、危急值 | interpretation_status、manual_review_reason | 满足复核条件时 interpretation_status=need_manual_review |

## 20. 报告类型处理规则

| report_classification | 是否进入指标抽取 | 是否进入异常判断 | 存储规则 |
|---|---|---|---|
| lab_report | 是 | 是 | 生成 report + item 明细 |
| imaging_report | 是 | 是 | 抽取影像征象字段 |
| pathology_report | 部分 | 部分 | 抽取结论型字段，默认 need_manual_review |
| surgery_record | 否 | 否 | 保存原文、摘要、附件；report_parse_status=unparsed 或 partially_parsed |
| discharge_summary | 部分 | 否 | 抽取诊断、用药、出院建议；不做检验异常比较 |
| prescription | 部分 | 否 | 抽取药名、剂量、频次；不做检验异常比较 |
| unknown | 否 | 否 | interpretation_status=need_manual_review |

## 21. 异常标记生成规则

| rule_key | 输入 | 输出 | 规则 |
|---|---|---|---|
| abnormal_from_original_flag | OCR 原始 H/L/↑/↓/偏高/偏低/阳性/阴性 | abnormal_flag | 高、H、↑、偏高映射 high；低、L、↓、偏低映射 low；危急值映射 critical_high/critical_low；阳性类结果映射 value_text=positive |
| abnormal_from_range_high | value_numeric、reference_high | abnormal_flag=high | reference_high 非空且 value_numeric > reference_high |
| abnormal_from_range_low | value_numeric、reference_low | abnormal_flag=low | reference_low 非空且 value_numeric < reference_low |
| abnormal_from_range_normal | value_numeric、reference_low、reference_high | abnormal_flag=normal | reference_low <= value_numeric <= reference_high |
| abnormal_from_text_positive | value_text | value_text=positive | 阳性、检出、有、+、1+、2+、3+、4+ 映射 positive |
| abnormal_from_text_negative | value_text | value_text=negative | 阴性、未检出、无、- 映射 negative |
| abnormal_unknown | 缺少数值、单位无法识别、参考范围无法解析且无原始异常标记 | abnormal_flag=unknown | interpretation_status=need_manual_review 或 partially_interpreted |

## 22. 解读等级规则

| interpretation_level | 条件 |
|---|---|
| normal | 已解析指标均为 normal/negative，且无 critical 标记 |
| mild_abnormal | 存在 high/low/positive，但无 critical，且不命中重点风险字段 |
| moderate_abnormal | 血尿酸未达标、肝肾功能异常、尿检异常、炎症指标异常、血糖血脂异常、影像阳性任一成立 |
| severe_abnormal | 多组异常同时存在，或肾功能明显异常、肝功能明显异常、感染支持证据阳性、尿酸盐沉积证据阳性 |
| critical | abnormal_flag=critical_high/critical_low，或 OCR 原文包含危急值/危急/危急报告 |
| unknown | report_parse_status=unparsed 或 interpretation_status=not_interpretable |

## 23. 人工复核触发规则

| rule_key | 条件 |
|---|---|
| review_low_confidence | ocr_confidence < 0.8 或 item_confidence < 0.8 |
| review_missing_reference | 数值型指标缺少 reference_range_text 且无原始异常标记 |
| review_unit_unrecognized | 数值型指标 unit 为空或无法识别 |
| review_item_unmapped | standard_key=unmapped |
| review_conflicting_result | 原始异常标记与参考范围比较结果冲突 |
| review_critical_value | abnormal_flag=critical_high/critical_low |
| review_unsupported_report_type | report_classification=surgery_record、unknown |
