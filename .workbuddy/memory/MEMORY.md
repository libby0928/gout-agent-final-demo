# 项目记忆

## 专家配置
- 默认专家：痛风产品助手（gout-product-assistant）
- 专家定位：痛风智能体产品设计师，覆盖原型设计、需求文档撰写、Demo页面迭代
- 安装路径：~/.workbuddy/plugins/marketplaces/my-experts/plugins/gout-product-assistant
- 内置 Skill：gout-design-system（包含设计规范、需求文档、AGENTS.md 等完整项目知识）
- 内置市场 Skill：impeccable（UI/UX 设计工具集 v2.0）

## 使用方式
在 WorkBuddy 中打开本项目后，从左侧"专家"面板选择"痛风产品助手"即可激活。建议在 WorkBuddy 界面中将本项目添加为"项目"并绑定该专家为默认专家。

## 患者端手机号规则
- 患者端手机号补充支持微信授权或手动输入两种路径。
- 凡手动填写的手机号，必须做验证（格式校验 + 接收并回填验证码/短信校验），未通过验证不允许进入后续建档或绑定流程；微信授权带回的手机号视为已验证，可不重复校验。

## 项目约定
- 所有页面按"可直接对外展示"的成品标准处理
- 界面不放内部说明、开发备注或演示口径
- 涉及主流程、关键状态、主任待确认事项的修改需先复述需求并确认后再动文件
- 患者端首页后续改版时，底部所有按钮必须保持当前状态不变，不修改样式、布局、功能、交互行为、页面位置、外观或点击响应

## 设计决策固化（2026-07-10）
- 患者端首页信息卡：源文件圆角浅底卡片形式；不展示进度条、不展示状态标签 pill（仅为 hero 标题语义色参考）；病历夹入口固定在信息卡右上角。
- 全页统一色板四档语义色：主渐变(青绿→蓝)、主青绿(#03D49F/#03A98A)、主蓝(#01A1FF/#0184CF)、警告橙(#ED7B2F/#FFAA44)；禁止引入第四种主题色。

## 医生端 Demo 设计决策（2026-07-13）
- 医生端 Demo 整体视觉**向患者端靠拢**：主色用青绿→蓝渐变 #03D49F→#01A1FF（即 tokens.css 的 --patient-* 变量），大圆角(18-26px)、宽松行高、装饰氛围同患者端。此决策推翻了早期令牌文档 v1.0「医生端刻意区别于患者端（蓝色工作台）」的红线。
- 仍保留的医生端特质：高密度列表/分块/来源标记、底部操作条（非患者端四胶囊导航）、状态色语义铁律（橙=待确认/绿=已确认/红=异常）、AI 辅助卡必标「诊疗辅助建议」、线性 SVG outline 图标。
- 基线文件：设计系统确认书 v2 `/design-system/医生端Demo设计系统确认书.md`；原型源文件 `/design-system/doctor-demo.html`；成品 `/demos/医生端Demo.html`（12 屏单文件，含双向联动 demoState 模拟、工作流分组导航、重置演示按钮）。
- 已知演示限制：二维码为演示伪码（非真实可扫）；联动为同文件页面内模拟（非与患者端 HTML 同运行域实时联调）；主动建档手机号未做格式校验。如需真·双向实时联动，需合并两端到同一运行域或加 postMessage 通信层。
