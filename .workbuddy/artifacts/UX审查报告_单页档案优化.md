# 医生端 Demo · UX 专项审查报告
## 审查目标：单页档案 + 极少点击

---

## 一、审查范围与方法

审查文件：`demos/医生端Demo.html`（2645 行单文件）
审查维度：
- 患者档案首屏信息完整度（能否"一页看完"做临床判断）
- 典型工作流点击次数（待接诊 → 档案 → 风险/用药/检验/发作 → 下一步）
- 冗余交互（重复入口、重复信息、不必要的 tab/展开/弹窗）

典型医生工作流模拟：
- **场景 A**：待接诊 → 判断是否需要纳管（看风险/用药/检验）
- **场景 B**：已纳管 → 评估用药是否需要调整（看用药方案 / 检验 / 发作）
- **场景 C**：任何患者 → 补录资料 → 查看归档结果

---

## 二、当前档案结构分析

`renderProfile(pid)` 渲染顺序（L1884-1901）：

```
1. visitNumber          — 本次就诊号 bar
2. patientInfoCard      — 姓名/性别/年龄/手机/状态 chip
3. conciseBrief         — AI 临床摘要（首屏核心）
   ├─ AI 临床摘要       — 分期 chip + 当前风险 + 警示 tags
   ├─ 关键指标4格       — 血尿酸 / 肾功能 / 近期发作 / 下一步
   ├─ 当前用药方案(简)  — 长期 + 急性 各一行（无约束）
   ├─ 既往史与安全      — 只显示前2条摘要
   └─ 操作按钮区        — 诊间接诊 / AI辅助治疗 / 补录资料
4. 档案操作栏           — "查看完整档案" + "档案维护"
5. [展开后] tabs        — 病史与发作 / 用药方案 / 检查与就诊
   └─ tab内容           — 完整用药方案(含约束) / 检验 / 发作记录
```

## 三、问题清单（按优先级）

### 🔴 P0 — 首屏信息缺失，迫使医生额外点击

#### P0-1：完整用药方案被折叠在 tab 后
- **现状**：首屏只展示简化用药（"非布司他40mg/秋水仙碱0.5mg"），合并症约束、调整依据、依从性/尿酸目标全部在"查看完整档案→用药方案 tab"后
- **影响**：医生看到高血压患者时，无法在首屏看到"合并高血压，监测血压"和"eGFR<45优选非布司他"这些关键安全约束
- **当前点击数**：首屏→查看完整档案(1)→用药方案 tab(1) = **2次额外点击**
- **建议**：将 `medicationPlanCard` 的完整内容（含合并症约束）直接内嵌到首屏 conciseBrief 中

#### P0-2：关键检验结果不在首屏
- **现状**：首屏"关键指标"区只显示"最近血尿酸 542 μmol/L ↑"和"肾功能 肌酐 78·正常"两行，但完整化验单（含 ALT 26 U/L 等）在"检查与就诊"tab
- **影响**：医生评估肝功能时需额外点击，而肝功能是降尿酸药物安全性监测的核心
- **当前点击数**：首屏→查看完整档案(1)→检查与就诊 tab(1)→找化验单 = **2-3次额外操作**
- **建议**：在首屏关键指标区直接展示血尿酸 + 肌酐 + ALT 三行（最重要的是异常项标红）

#### P0-3：发作记录细节需要 tab 切换
- **现状**：首屏只显示"近期发作 >5 次"，具体发作部位、严重程度、处理方式全在"病史与发作"tab
- **影响**：医生决定是否启动/调整降尿酸治疗时，发作频率和严重程度是核心依据
- **建议**：在首屏直接展示最近1-2次发作的日期+部位+程度（无需点击展开）

---

### 🟡 P1 — "单页档案"概念矛盾，交互冗余

#### P1-1："查看完整档案"按钮与"单页看完"目标矛盾
- **现状**：`archiveExpanded` 机制暗示首屏是"摘要"，"完整"内容需要展开
- **影响**：这与用户"一面一页看完"的北极星直接冲突
- **建议**：取消 archiveExpanded 的折叠机制 → 所有信息直接在首屏纵向排列 → 删除"查看完整档案"按钮 → 保留"档案维护"（进入编辑模式）

#### P1-2：三个 tab 需要手动切换
- **现状**：病史与发作 / 用药方案 / 检查与就诊 三个 tab 互斥显示
- **影响**：医生查看用药后想看检验，必须再点一个 tab。检验和用药不是互斥的——它们需要一起看
- **建议**：取消 tab 互斥机制 → 改为三个连续的 section 标题（纵向排列，全部可见，滚动直达）

#### P1-3：重复的用药方案展示
- **现状**：首屏有简化用药方案卡片；展开后"用药方案"tab 有完整 `medicationPlanCard`（含合并症约束）。两份内容重复且用户需要点两次
- **建议**：统一成一个完整的 medicationPlanCard，直接放首屏

#### P1-4：资料补充入口分散
- **现状**：
  - 首屏操作区有"补录资料"
  - 底部有"档案维护"
  - 患者列表每行有"补录资料"
  - 这些入口行为不完全相同
- **建议**：统一为 1 个"补充/维护资料"入口，减少医生选择成本

---

### 🟢 P2 — 细节打磨

#### P2-1：既往史与安全约束对用药决策至关重要
- **现状**：conciseBrief 只显示"共4条"和前2条摘要
- **建议**：对于有合并症（高血压/糖尿病/肾结石）的患者，直接在首屏展示对应的安全约束（用药禁忌），这是给医生最重要的决策辅助

#### P2-2："诊间接诊"按钮状态不够智能
- **现状**：所有患者档案都显示"诊间接诊"
- **建议**：已纳管/长期管理中显示"复诊评估"；逾期患者显示"联系患者"或"安排复诊"；首次就诊显示"开始接诊"。让按钮反映当前最需要的下一步

#### P2-3：补录资料后无视觉定位
- **现状**：化验单归档后 `archiveExpanded=true`，但医生需要手动滚动到资料区
- **建议**：归档后自动平滑滚动到新增资料位置，新资料卡片短暂闪烁/高亮 2 秒

#### P2-4："AI 辅助治疗" 跳转到独立屏
- **现状**：点击后跳转到 `screen-aiassist`（独立页面），打断档案上下文
- **建议**：改为在档案页面内以 bottom sheet / 展开卡片形式展示 AI 建议，保持档案上下文不丢失

## 五、改造方案概要

### 核心改造：重新设计 `renderProfile(pid)` 的布局顺序

**新首屏结构（一条流看完，无需点击展开）：**

```
┌─ 本次就诊号 bar ──────────────────────┐
│  [就诊号] [修改/录入]                    │
├─ 患者信息卡片 ──────────────────────────┤
│  姓名 + 状态chip | 性别·年龄 | 手机     │
├─ AI 临床摘要（分期 + 风险 + 警示）────┤
│  急性发作期 · 右足红肿热痛               │
│  ⚠ 尿酸未达标 ⚠ 服药不规律 ⚠ 缺肝肾功能 │
├─ 关键检验指标（3行完整展示）─────────────┤
│  血尿酸 542 μmol/L ↑  |  肌酐 78 正常   │
│  ALT 26 U/L |  最近报告 2026-07-08      │
│  [查看化验单原件]                         │
├─ 长期降尿酸方案（完整版含约束）──────────│
│  方案 | 依从性 | 调整依据 | 合并症约束   │
│  [调整方案]                               │
├─ 急性发作处理方案（完整版含约束）────────│
│  方案 | 启用条件 | 注意事项 | 合并症约束  │
├─ 最近发作记录（最近2条）────────────────│
│  07-08 · 第一跖趾关节 · 中度疼痛          │
│  06-15 · 第一跖趾关节 · 轻度 · 自行缓解  │
│  [查看全部发作记录]                       │
├─ 既往史与安全约束 ──────────────────────│
│  高血压·规律服药 · 监测血压               │
│  药物过敏·无 · 肾结石·无                 │
│  家族史·不详                              │
├─ 补录资料区 ─────────────────────────────│
│  07-14 · 门诊病历 · 待确认                │
│  [补充资料]                               │
├─ 操作区 ─────────────────────────────────│
│  [复诊评估] [AI 辅助建议] [档案维护]      │
└──────────────────────────────────────────┘
```

### 关键改动点：

1. **删除 `archiveExpanded` 机制** → 所有内容直接渲染
2. **删除 tab 互斥** → 改为连续 section 标题
3. **删除"查看完整档案"按钮** → 页面本身就是完整的
4. **medicationPlanCard 完整版直接放首屏** → 含合并症约束
5. **关键检验直接在首屏展示 3 行** → 血尿酸/肌酐/ALT
6. **最近发作记录直接展示** → 最近2条关键信息
7. **操作按钮根据患者状态动态显示** → 智能下一步
8. **"AI 辅助治疗"改为页内 bottom sheet** → 不跳出档案上下下文
