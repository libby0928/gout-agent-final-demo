# 痛风专病档案字段完整版

本文档用于整理通用智能体档案和痛风专病智能体档案中需要采集、展示、识别和归档的字段。字段来源包括既有痛风专病档案字段、检验检查字段，以及本次截图中的健康报告、体格检查、问卷、用药和上传报告字段。本文档只保留产品和业务讨论可直接使用的字段口径：一级目录、二级目录、字段、取值方式/可选值、参考范围/判定标准、展示原则、确认要求。

一级目录按字段归属区分：“通用智能体档案”承接跨病种通用的个人资料、生活方式、既往史、体格检查、用药、检验检查和报告资料；“痛风专病智能体档案”承接痛风专病特有或在痛风管理中具有明确专病意义的病史、尿酸目标、痛风证据、专病随访和专病资料；“小程序”承接患者端页面流程、状态流转和端侧操作字段；“医生端”只作为患者端状态来源、资料确认和回写依据，不作为患者端展示承载面。

“取值方式/可选值”不等同于全部字段都只能枚举。写成固定选项时，表示前端可优先提供快捷选项；写成“选项 + 其他可填”时，表示既有推荐选项，也允许患者、医生或 OCR 写入其他内容；写成“文本”“数值 + 单位”“日期/时间”“文件上传/图片上传”“文本/系统生成”时，表示该字段不是封闭枚举，应按实际内容存储。

“参考范围/判定标准”用于承接截图中出现的参考范围、报告原文正常值、阴阳性判定或专病目标值。截图中大量 `0-1000` 属于当前原型示例范围，不应直接当作真实医学正常值；正式上线时应优先使用医院 LIS/HIS 报告原文、OCR 识别出的参考范围和医生确认后的专病目标。

“展示原则”以患者端定版信息架构为准，不再按字段重要性泛化展示。患者端只承接患者当下需要看到或操作的内容：未建档首页、快速建档页、建档中首页、已纳管首页、我的病历夹首页、病历夹二级详情/查看依据、复诊闭环、扫码就诊、联系医生、上传报告/附件面板、健康打卡/饮食问答。完整结构化字段、医生确认过程、后台规则和不适合患者直接查看的判断信息，不在患者端平铺展示，保留在医生端确认页或后台配置中。

快速建档仍只保留姓名、性别、手机号；其他字段进入通用档案、痛风专病档案、诊前补充、医生确认、报告识别或随访管理流程。
