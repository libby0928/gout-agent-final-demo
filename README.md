# 痛风智能体

这是痛风专病智能体 Demo 的本地仓库，存放患者端、医生端、业务规则、字段字典、视觉规范和可复用设计组件。项目默认按可直接对外展示的原型标准维护：界面只放患者或医生当下需要看到的内容，流程规则和设计说明沉淀在文档里。

## 当前主资产

- `痛风智能体患者端_2026年7月3号（会议后修订版）.html`：患者端主 Demo，包含阿福式首页、扫码建档、病史补充、报告上传、复诊闭环、病历夹等状态。
- `医生端_信息降噪版Demo.html`：医生端接诊采集版 Demo，主线围绕待接诊、诊间采集、字段确认、写入档案和随访计划。
- `痛风智能体Demo设计规范_AI执行版.md`：当前最完整的业务、交互、视觉和组件规则。
- `docs/99-索引/当前工作台账_20260708.md`：当前对话和文件台账，区分主线文件、历史文件、试验稿和本地输出。
- `docs/02-字段字典/痛风专病智能体患者端字段字典_v1.1.1_20260706.md`：患者端研发字段字典。
- `痛风专病档案字段梳理.md`、`痛风相关检验检查字典.md`：结构化字段和检验检查补充字典。
- `design-system/`：可复用视觉令牌、组件样式和组件示例。
- `docs/`：仓库结构和复用规则说明。

`痛风专病智能体 demo-修改版.html` 和 `痛风专病智能体 demo.html` 保留作历史参考，不作为新的默认修改入口。

## 复用方式

新做页面或改现有 Demo 前，先看 `docs/复用规则.md`，再从 `design-system/components.css` 复制或引用对应组件。患者端优先复用阿福式对话容器、底部输入区、快捷胶囊、档案完整度卡片和病历夹组件；医生端优先复用工作台任务卡、待办列表、状态标签、字段确认卡和证据来源样式。

对外展示页面不要出现开发说明、演示说明、字段 ID、内部规则解释或占位备注。这些内容应放入规范文档或对话说明。

## Git 使用

当前仓库默认分支是 `main`。本地工具缓存、飞书临时二维码和 `output/` 备份产物已通过 `.gitignore` 排除；正式 HTML、文档和素材会进入版本库。
