# 医生端 Demo · 交付说明

> 面向演示者的极简说明。本说明不写入 HTML 页面，仅供演示前快速查阅。

## 一、文件清单

| 文件 | 说明 |
|------|------|
| `医生端Demo.html` | 医生端单文件演示原型（自包含，浏览器直接打开即可运行，无需联网、无需服务器） |
| `医生端Demo_交付说明.md` | 本说明文档 |

> 说明：原型内建档码优先引用仓库素材 `../assets/doctor-qr.png`；若打开环境无法解析该相对路径，会自动回退为内嵌 SVG 占位码，不影响演示运行。

## 二、12 屏清单

1. 登录 / 工作台入口（screen-login）
2. 工作台总览（screen-workbench）
3. 患者列表（screen-patients）
4. 患者专病档案（screen-profile）
5. 看诊前资料（screen-material）
6. AI 诊疗辅助（screen-aiassist）
7. 接诊确认（screen-confirm）
8. 复诊逾期管理（screen-overdue）
9. 二维码 / 建档码入口（screen-qrcode）
10. 语音诊间记录（screen-voice）
11. 就诊码 / 接诊扫码（screen-visitqr）
12. 医生主动建档（screen-enroll）

## 三、如何打开演示

- 直接双击 `医生端Demo.html`，用任意现代浏览器（Chrome / Edge / Safari / Firefox）打开即可。
- 无需安装依赖、无需启动本地服务、无需联网。
- 页面为「iPhone 设备外壳 + 右侧工作流导航」的展示布局；点击右侧导航项或屏内按钮即可切换 12 个屏幕。

## 四、演示联动入口

- **右下角「患者端联动（演示）」面板**：点击展开后可模拟触发患者端行为（如患者补资料、到院扫接诊码），并实时联动医生端对应屏幕的状态变化，用于演示医患双向协同。
- 面板内三个按钮：
  - 患者补充病史 / 报告 → 联动「看诊前资料」「工作台」状态
  - 患者到院扫接诊码 → 联动「就诊码 / 接诊扫码」状态
  - 患者建档（演示） → 联动「工作台待接诊」角标

## 五、重置演示

- 右侧工作流导航区最底部有**「重置演示」**按钮，点击后清空所有联动状态，回到初始登录 / 工作台状态，便于重复演示。

## 六、已知演示限制

1. **二维码为演示伪码**：建档码优先展示素材图，就诊码由 JS 实时生成的伪 QR 风格图形，**不可真实扫描**，仅作视觉演示。
2. **联动为同文件页面内模拟**：患者端联动全部在同一 HTML 文件内通过 JS 状态切换模拟，并非真实跨端通信。
3. **主动建档手机号未做格式校验**：第 12 屏「医生主动建档」中的手机号输入框仅作演示录入，未做号码格式 / 合法性校验。
4. **状态为内存态**：所有演示状态保存在页面内存中，刷新页面即重置为初始种子数据（等同于点击「重置演示」）。
